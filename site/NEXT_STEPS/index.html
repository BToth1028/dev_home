    üöÄ Step 1: Create GitHub Repository¶

    Option A: Via GitHub Web UI 1. Go to https://github.com/new 2. Name: engineering-home (or dev-home) 3. Visibility: Public (recommended) or Private 4. DO NOT initialize with README, .gitignore, or license (you already have these) 5. Click "Create repository" 6. Copy the repository URL

    Option B: Via GitHub CLI 

    gh repo create engineering-home --public --source=. --remote=origin --push

    üöÄ Step 2: Push to GitHub (Manual Method)¶

    If you created the repo via web UI:

    # Add the remote (replace YOUR_USERNAME)
git remote add origin https://github.com/YOUR_USERNAME/engineering-home.git

# Push to GitHub
git push -u origin main

    Verify: 

    git remote -v

    üöÄ Step 3: Set Up Branch Protection¶

    On GitHub: 1. Go to your repo ‚Üí Settings ‚Üí Branches 2. Add branch protection rule for main: - ‚òë Require a pull request before merging - ‚òë Require status checks to pass before merging (when you add CI) - ‚òë Include administrators (if you want to follow your own rules) 3. Save changes

    üöÄ Step 4: Update CODEOWNERS¶

    Edit .github/CODEOWNERS and replace YOUR_GITHUB_USERNAME with your actual GitHub username:

    # Global fallback
* @your-actual-username

    Then commit and push: 

    git add .github/CODEOWNERS
git commit -m "chore: update CODEOWNERS with actual username"
git push

    üéØ Step 5: Try Bootstrapping a New Project¶

    Test the Python API template:

    # Bootstrap a new project
Copy-Item -Recurse -Force C:\dev\templates\starter-python-api C:\dev\apps\test-api
cd C:\dev\apps\test-api

# Initialize as separate repo
git init
git add .
git commit -m "feat: bootstrap from engineering-home template"

# Run it locally
cp .env.example .env
docker compose up --build

    Visit http://localhost:8000 to see it running!

    Clean up test: 

    cd C:\dev
Remove-Item -Recurse -Force C:\dev\apps\test-api

    üîß Optional: Add CI/CD to Engineering Home¶

    Create .github/workflows/validate-templates.yml:

    name: Validate Templates
on: [push, pull_request]
jobs:
  validate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: List templates
        run: ls -la templates/
      - name: Validate markdown
        run: find . -name "*.md" -exec echo "‚úì {}" \;

    Commit and push: 

    git add .github/workflows/validate-templates.yml
